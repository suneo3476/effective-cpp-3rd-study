# Item 1: C++を言語の連合体と見なす

## 核心
C++は1つの言語ではなく、4つのサブ言語の集合体。

## 4つのサブ言語

| サブ言語 | 特徴 | 値渡し/参照渡し |
|---------|------|----------------|
| C | 組み込み型、ポインタ、配列 | 値渡し |
| Object-Oriented C++ | クラス、継承、仮想関数 | const参照渡し |
| Template C++ | ジェネリック、TMP | 型による |
| STL | コンテナ、イテレータ | 値渡し |

## ポイント
- 「C++のベストプラクティス」は存在しない
- 今どのサブ言語で書いてるかで判断が変わる
- 矛盾するアドバイス（「参照で渡せ」「値で渡せ」）は、文脈が違う

## 自分メモ
- 普段はあまり意識せず、習った時の書き方をまねてた
- どのサブ言語か意識すると、ルールの使い分けが明確になる

## 覚えておくこと
> C++の効果的なプログラミングのルールは、使っているC++の部分によって変わる。
